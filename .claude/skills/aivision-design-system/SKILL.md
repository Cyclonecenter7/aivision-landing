# AIVISION · Дизайн-система

AIVISION — российский B2B-консалтинг для собственников бизнеса (оборот от 10 млн ₽/мес). Строит «системы управляемой прибыли»: находит где теряется маржа, внедряет KPI и управленческий учёт, собирает единый BI-центр принятия решений.

> «Мы не даём советы. Мы строим систему и ведём бизнес к результату.»

**Две поверхности бренда:**

| Поверхность | Контекст | Характер |
|---|---|---|
| **Маркетинг** | Лендинг, кейсы, слайды, рекламные материалы | Крупная типографика, воздух, одна CTA на блок |
| **Дашборд / BI** | Панель собственника, отчёты, BI-аналитика, внутренние инструменты | Плотная верстка, tabular-nums, 3 data-цвета |

---

## 1 · Когда применять скилл

Использовать когда пользователь просит сделать что-то в бренде AIVISION:
- Секции лендинга, страницы кейсов, питч-деки, рекламные креативы, письма
- Копирайт в стиле AIVISION (русский язык, диагностический тон)
- Дашборды, панели собственника, BI-отчёты, CRM-экраны, внутренние инструменты
- Любые интерфейсы продукта AIVISION

НЕ применять для дизайна других брендов или на английском.

## 2 · Что читать перед работой

**Всегда**, перед тем как начать:

1. `resources/colors_and_type.css` — токены и утилиты. Всегда подключать.
2. `resources/REFERENCE.md` — полная документация (контент, визуал, иконография). ~200 строк — просматривать каждый раз.
3. **Маркетинг:** `resources/ui_kits/marketing/` — React-компоненты лендинга. Копировать, не переписывать.
4. **Дашборд:** `resources/ui_kits/dashboard/tokens.css` + `resources/ui_kits/dashboard/Primitives.jsx` — все атомы дашборда.

Визуальные примеры каждого токена — в `resources/preview/` (один HTML-файл на каждый компонент).

## 3 · Пять правил, которые нельзя нарушать

1. **Chamfer снизу справа на каждом интерактивном/контейнерном элементе.**
   ```css
   clip-path: polygon(0 0, 100% 0, 100% calc(100% - Xpx), calc(100% - Xpx) 100%, 0 100%);
   ```
   Классы: `.av-chamfer-sm` (10) `-md` (12) `-lg` (16) `-xl` (20) `-2xl` (28).
   На дашбордах — меньше: 4 / 6 / 8 / 10 / 12 / 16 / 20.

2. **Никаких `border-radius`** — кроме настоящих кругов (буллеты, прогресс-бары, аватары).

3. **Никаких градиентов, glassmorphism, теней** — кроме `shadow-sm` на hover-карточках и синего glow `0 0 32px 8px rgba(63,110,232,.25)` под hero-акцентом.

4. **Один акцент на блок.** `#3F6EE8` встречается один раз. На синем фоне акцент становится белым.

5. **Каждая секция начинается с eyebrow** — 24px цветная линия + 12px UPPERCASE-лейбл, `letter-spacing: +0.15em`. Цвет кодирует смысл: синий = продукт/дефолт, красный = проблема, чёрный = сравнение/вывод.

## 4 · Голос бренда — как пишет AIVISION

- **Только русский язык.** Деловой тон, никакого маркетингового хайпа.
- **От первого лица множественного числа («мы»).** К читателю — косвенно: *«Собственник видел оборот, но не видел маржи»*, не *«Вы увидите свою маржу»*.
- **Никаких восклицаний. Никаких эмодзи. Никаких слов-усилителей** («уникальный», «революционный», «гарантированно»).
- **Диагностический триптих** — каждый блок: *проблема → механизм → разрешение*. Тире вместо глаголов-связок: *«Доход рос — расходы быстрее.»*
- **Числа — визуальная валюта.** Всегда со знаком: `+93%`, `−32%`, `200М+`. Настоящий минус `−` (U+2212), не дефис. Валюта: `₽` постфиксом с неразрывным пробелом: `376 185 ₽`.
- **Регистр:** заголовки — сентенс-кейс, последнее слово часто в `var(--av-blue)`. Эйбровы — UPPERCASE.
- **Эйбровы:** ПРОБЛЕМА · ПРОДУКТЫ · КЕЙСЫ · ДИАГНОСТИКА · СИСТЕМА УПРАВЛЯЕМОЙ ПРИБЫЛИ.
- **CTA:** «Разобрать бизнес» · «Начать диагностику» · «Узнать подробнее».

## 5 · Токены (быстрый справочник)

```
/* Бренд */
--av-blue:        #3F6EE8     /* единственный акцент */
--av-blue-hover:  #2f5dd2
--av-blue-100:    #F0F3FA     /* тинт для result-карточек */
--av-black:       #0A0A0A     /* основной текст + тёмные поверхности */
--av-ink-900:     #181818     /* тёмная карточка / модалка */
--av-red:         #E5484D     /* только для проблем */

/* Светлые поверхности */
--av-surface-100: #F4F4F5     /* фон страницы */
--av-surface-200: #F0F2F5     /* альт фон */
--av-border-200:  #E8E8E8     /* дефолтный бордер */

/* Типографика */
--av-font-sans:   'Inter'     /* единственная гарнитура, 400–900 */
```

**Chamfer (маркетинг):** 10 / 12 / 16 / 20 / 28.
**Chamfer (дашборд):** 4 / 6 / 8 / 10 / 12 / 16 / 20.
Паддинг секций: `py-16 md:py-24`. Контейнер: `max-w-6xl`. Сетка: `grid-cols-3 gap-5`.

## 6 · Компоненты — маркетинг

| Паттерн | Рецепт |
|---|---|
| **Primary кнопка** | `bg:#3F6EE8 → #2f5dd2` hover, белый текст, chamfer 16, padding `12 24`, 14px 500 |
| **Dark кнопка** | `bg:#0A0A0A → #3F6EE8` hover. Навбар. |
| **Invert кнопка** | белый bg на синей поверхности, `→ #EEF3FD` hover |
| **Светлая карточка** | `#fff` + `1px #E8E8E8` + chamfer 20 + padding 24–32 |
| **Тёмная карточка** | `#181818` + `1px #2A2A2A` + chamfer 28 |
| **Result-карточка** | `#F0F3FA` + `1px #E4E8F0` + chamfer 10–12 (вложена в продуктовую) |
| **Числовой бейдж** | 40×40, chamfer 12. Варианты: синий/белый/чёрный |
| **Метрика** | большое число 700–900, 28–72px, `letter-spacing:-0.02em` + UPPERCASE лейбл сверху |
| **Eyebrow** | 24px линия + 10px gap + 12px UPPERCASE 500 `tracking-widest` |

## 7 · Компоненты — дашборды

Подключить `resources/ui_kits/dashboard/tokens.css` + `Primitives.jsx` → все атомы доступны через `window.DB.*`.

**Три data-цвета — только они, никаких других:**
- `#16A34A` зелёный = прибыль, рост, закрытая сделка, положительная дельта
- `#E5484D` красный = убыток, падение, перерасход, отрицательная дельта
- `#3F6EE8` синий = выручка, объём, нейтральная метрика

Янтарный `#EAB308` — **только для статус-тегов в пайплайне**, никогда для KPI-чисел.

| Атом | `window.DB.*` | Когда использовать |
|---|---|---|
| KPI-карточка | `DB.Kpi` (+ `compact` / `hero`) | Страница всегда начинается с 3–6 KPI |
| Якорная полоса | `DB.KpiStrip` | Одна доминирующая цифра + 3 вторичных |
| LIVE-тикер | `DB.Ticker` | Чёрная полоса в самом верху экрана |
| Editorial-заголовок | `DB.EditorialHead` | Заголовок-гипотеза («Октябрь. Узкое место — квалификация.») |
| Диаг. колонка | `DB.DiagColumn` | Чёрная правая колонка с 3 диагностическими узлами |
| Callout | `DB.Callout` | Чёрная полоса между KPI и таблицей — вывод AIVISION-тоном |
| Таблица | `DB.Table` + `DB.Delta` | Любые строчные данные (сделки, продукты, операции) |
| Прогресс-бар | `DB.Bar` | Inline в таблице, категорийный breakdown |
| Воронка | `DB.Funnel` | Этапы конверсии, можно помечать «УЗЕЛ» |
| Пайплайн | `DB.PipelineColumn` + `DB.StageCard` | Kanban-стадии сделок |
| Линейный чарт | `DB.LineChart` | Временной ряд; факт + план-пунктир |
| Sparkline | `DB.Sparkline` / `DB.SparkMini` | Inline в заголовке карточки или ячейке таблицы |
| Фильтр-бар | `DB.FilterBar` / `DB.PeriodPicker` | Над каждой таблицей / дашбордом |
| Сайдбар | `DB.Sidebar` | Только на продуктовых экранах |

**Плотность:** строка таблицы 32px, padding ячейки 10×16. Лейбл 10px UPPERCASE. Тело 12–13px. Шкала метрик 16 / 22 / 28 / 48 / 72 / 96px.

**Числа:** всегда `font-variant-numeric: tabular-nums`. Дельты со знаком, настоящий минус U+2212. Валюта `₽` постфиксом.

**PII:** имена клиентов в мок-демо — класс `.db-pii` (`filter: blur(4px); user-select:none`).

**Графики:** SVG-полилинии по умолчанию — см. `DB.LineChart`. Recharts подключать только если нужна интерактивность; `chartConfig = { positive: '#16A34A', neutral: '#3F6EE8', negative: '#E5484D' }`.

## 9 · Иконографика

- **Библиотека:** [Lucide](https://lucide.dev). Только stroke, `stroke-width: 2`, размер 20px (12–16 внутри кнопок).
- **Никаких fill, emoji, иконочных шрифтов.** Цвет через `currentColor`.
- **Позиционирование:** слева от лейбла в result-блоках, справа от текста в CTA.
- **Частые иконки:** `BarChart2`, `Target`, `TrendingUp`, `Diamond`, `Shield`, `Handshake`, `Settings`, `ArrowRight`, `ArrowLeft`, `X`.

## 10 · Рабочий процесс

1. Прочитать `resources/REFERENCE.md`.
2. Определить поверхность — маркетинг или дашборд.
3. **Маркетинг:** скопировать `resources/ui_kits/marketing/index.html` как scaffold. Секции: `Navbar → Hero → Problem → Products → Cases → ComparisonWithForm → Footer`. Порядок загрузки скриптов: `Primitives.jsx` первым.
4. **Дашборд:** скопировать `resources/ui_kits/dashboard/index.html`. Подключить `tokens.css` после `colors_and_type.css`. Использовать атомы из `DB.*`.
5. Для разового артефакта (слайд, письмо, реклама) — вставить только нужное из `Primitives.jsx`.
6. Всегда подключать `resources/colors_and_type.css`.

## 11 · Можно / Нельзя

| Можно | Нельзя |
|---|---|
| Плоские цвета, один акцент на блок | Градиентные фоны, мультистоп-заливки |
| Chamfer снизу справа на всём | Скруглённые углы на кнопках/карточках |
| Только Inter, 400–900 | Смешанные гарнитуры, серифы в заголовках |
| Большие жирные числа как якоря | Иконочный «data slop» — куча иконок вместо данных |
| Lucide stroke, размер 20, weight 2 | Filled-иконки, эмодзи, разные размеры |
| `−93%` с U+2212 | `-93%` с дефисом |
| Одна чёткая CTA, реальный контент | Заглушки, lorem, несколько CTA подряд |
| `.db-pii` для имён клиентов в мок-демо | Придуманные реальные имена |
